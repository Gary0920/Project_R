# Project_R 工作台 UI 重设计原型说明

## 目的

本原型用于在真实前端改造前，确认 Project_R 的整体风格、布局、通知系统、深浅色主题和客户情报面板设计方向。

## 文件结构

```
workbench-redesign/
├── index.html          # 入口选择页
├── variant-a.html      # 变体 A：经典三栏 · 对话优先
├── variant-b.html      # 变体 B：高密度 · 情报主舞台
├── shared/
│   ├── styles.css      # 公共样式、组件、深色主题
│   └── icons.svg       # 图标 Sprite
└── README.md           # 本文件
```

## 两个变体的核心差异

| 维度 | 变体 A（推荐） | 变体 B（探索） |
|---|---|---|
| **布局哲学** | 对话为中心，情报在侧边辅助 | 情报可升至主舞台，与对话并列切换 |
| **Sidebar** | 260px 经典宽度，信息舒展 | 200px 紧凑，释放中间空间 |
| **客户情报** | 320px 右侧面板内 Tab 切换 | 中间区域全宽覆盖层（540px+），更舒展 |
| **右侧面板** | 常驻，可同时展示文件/情报 | 只保留文件/来源，情报移出 |
| **通知中心** | 右上角下拉面板（400px） | 右侧滑入抽屉（380px），更像现代 App |
| **输入区** | 圆润大输入框，工具按钮文字+图标 | 扁平紧凑，默认图标-only |
| **强调色** | 柔和浅绿（继承现有 UI token） | 靛蓝，更冷峻 |
| **卡片风格** | 细边框 + 浅色背景 + 圆角 | 无边框 + 分隔线 + 扁平 |
| **信息层级** | 用背景和边框分隔层次 | 用留白和字号分隔层次 |

## 变体 A 设计取舍

**继承点：**
- 延续 Proma 布局节奏和 Project_R 用户已有的肌肉记忆
- 绿色强调色与现有 `docs/design/ui-design-language.md` 中 `--control-active-*` token 一致
- 三栏布局下，用户可以同时看到"对话、文件、情报"三重视角

**改进点：**
- 通知中心从旧版左下角Popover改为右上角独立面板，不再和用户头像挤在一起
- 客户情报面板明确区分"普通用户可读的业务视图"和"管理员高级操作区"
- GBrain 技术术语全部映射为业务语言（如"source"改为"资料来源"）
- Agent 执行流程增加"任务理解 → 计划 → 步骤 → 结果"的卡片化展示

**风险：**
- 客户情报挤在 320px 右侧面板内，图谱和时间线可能显得局促
- 三栏同时展开时，中间对话区在 1366px 屏幕上可能偏窄

## 变体 B 设计取舍

**突破点：**
- 客户情报不再作为"聊天旁边的参考资料"，而是可以作为独立主视图全宽展示
- 这更符合"CRM 客户情报中心"的产品定位——客户情报本身就是业务核心，不是聊天的附属品
- 紧凑布局和扁平风格更适合高密度信息展示（表格、图谱、时间线）

**继承点：**
- 仍保留左侧导航 + 中间工作区的大框架
- 聊天输入区仍固定在底部，不改写用户聊天习惯
- 深色模式逻辑与变体 A 一致

**风险：**
- "对话"与"情报"切换需要一次点击，不能像变体 A 那样一眼同时看到
- 扁平风格在长时间办公场景下可能视觉疲劳
- 与现有 Proma 参考差异较大，前端重构成本略高

## 推荐结论

**推荐采用变体 A 进入真实前端改造。**

理由：

1. **迁移成本最低**：变体 A 的三栏布局与现有 `frontend/src/renderer` 的 `AppShell`、`AppPage`、`ChatConversationPane`、`WorkspaceFilePanel` 结构完全对应，可以逐步替换样式而不改动布局架构。
2. **用户习惯延续**：Project_R 老用户已经适应"左侧会话 + 中间聊天 + 右侧文件"的三栏节奏，变体 A 只是在这个骨架上做视觉升级和信息重组。
3. **客户情报可后期升级**：变体 B 的"情报主舞台"概念很好，但可以在变体 A 的基础上通过"点击情报 Tab 后面板自动加宽到 480px"或"双击情报面板进入全宽模式"来渐进实现，不需要一次推翻布局。
4. **通知中心改进已验证**：变体 A 的通知中心从旧版左下角移到右上角独立空间，这是明确的需求改进，不需要依赖变体 B 的侧抽屉也能达成。
5. **深色模式更稳健**：变体 A 的卡片化层级在深色模式下更容易保持可读性；变体 B 的扁平分隔线在暗色主题中容易糊成一片。

**变体 B 的取舍可吸收为 A 的局部增强：**
- 变体 B 的"紧凑侧栏"可作为 A 的"侧栏收起态"（点击收起按钮后 Sidebar 缩为 48px 图标栏）
- 变体 B 的"扁平输入区"可作为 A 的响应式压缩态（窗口变窄时工具按钮自动收为图标）
- 变体 B 的"情报全宽"可作为右侧面板的"最大化"动作

## 待真实数据确认的占位

原型中以下入口保留了功能占位，标注为"待真实数据确认"：

- 设置页：服务器配置、提示词管理、Agent 行为边界、快捷键绑定
- 客户情报"关系图谱"Tab：当前为 CSS 模拟网络图，真实实现需接入 GBrain Graph API
- 客户情报"GBrain 状态"Tab：部分状态指标为静态值
- 搜索弹窗：搜索结果为假数据
- 管理员后台：用户管理表格、审计日志筛选、模板状态为静态数据
