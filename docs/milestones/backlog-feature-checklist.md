# 后补功能 Checklist

本文档用于管理 Project_R 当前“已识别但不阻塞本地使用”的后补功能。这里的功能不纳入当前本地必须通过项；当某项进入开发时，应先补充验收标准，再实现代码，最后从本文档迁移到根目录开发流程或 `docs/validation/electron-manual-checklist.md` 的正式验收区。

当前产品范围、阶段状态和业务 Skill 清单已从 `references/` 移到仓库根目录：

- `docs/product/Project_R PRD.md`
- `docs/milestones/Project_R 开发流程.md`
- `docs/product/Project_R 业务工作流清单.md`

## 使用规则

- [ ] 开发前确认该功能仍然符合当前产品方向。
  - 通过：功能仍解决真实使用问题，且不破坏 Project_R 后端中心化、安全受控的架构。
  - 失败：只是复刻 Proma 表象，或会引入不必要的客户端执行链路。
- [ ] 开发前补齐验收标准。
  - 通过：每项功能至少有“触发入口、成功状态、失败状态、权限边界、测试方式”。
  - 失败：只有功能名，没有判断完成的标准。
- [ ] 开发后更新主测试文档。
  - 通过：功能完成后迁移到根目录开发流程或 Electron 手工验收清单，并从本文档移除或标记完成。
  - 失败：功能已经实现但仍停留在后补池，导致测试范围不清。

## 1. 知识库与引用体验

- [ ] 引用来源点击打开原文标签页。
  - 当前状态：后补。
  - 目标：AI 回复中的来源可点击，并在 Project_R 内新建标签页查看原文。
  - 触发入口：知识库回答中的来源标记或来源列表。
  - 通过：点击来源后打开只读原文标签页，能看到文件名、路径/来源、正文内容。
  - 失败：点击无响应、打开错误文件，或没有权限仍能查看。
- [ ] 引用来源缺失时给出明确降级说明。
  - 当前状态：后补。
  - 目标：避免用户误以为所有回答都有可追溯来源。
  - 通过：无来源时显示“未命中知识库来源”或同等说明。
  - 失败：无来源但表现为知识库确定回答。
- [ ] 知识库刷新结果可视化更完整。
  - 当前状态：后补。
  - 目标：管理员能看到刷新耗时、文件数量、失败文件和最后更新时间。
  - 通过：刷新成功/失败都有可理解状态。
  - 失败：只显示成功/失败，没有定位信息。

## 2. 工作区文件能力

- [ ] 工作区文件预览。
  - 当前状态：后补。
  - 目标：用户可在工作区文件树中预览 Markdown、文本、PDF、Office 文件的只读内容或基础元信息。
  - 触发入口：Agent 工作区文件目录。
  - 通过：点击文件后可查看只读预览或基础元信息。
  - 失败：点击无响应，或下载/预览权限混乱。
- [ ] Office / PDF 项目资料解析。
  - 当前状态：后补。
  - 目标：工作区上传的 docx、xlsx、pptx、pdf 能解析为可检索文本，进入项目资料上下文。
  - 触发入口：工作区文件上传、项目资料刷新。
  - 通过：上传 Office/PDF 后索引状态可刷新，并能在当前项目对话中召回相关内容。
  - 失败：只保存文件但无法解析，或解析失败无提示。
- [ ] 异步项目 RAG 队列与真实向量化项目子库。
  - 当前状态：后补。
  - 目标：工作区资料变更后进入异步索引队列，项目对话检索当前项目独立向量库。
  - 触发入口：工作区文件管理区、管理员知识库区域。
  - 通过：资料变更后索引状态可见，刷新成功后只影响当前项目，其他项目和无项目会话不可召回。
  - 失败：项目资料混入全局知识库，或资料已变更但知识库仍使用旧内容且无提示。
- [x] 工作区文件回收区与恢复。
  - 当前状态：已完成 P9 竖切片。
  - 已实现：多文件上传、100MB 单文件限制、删除进入回收区、恢复/永久删除、成员/管理员权限、审计日志、项目索引状态 pending/indexed。
  - 后续增强：文件预览、Office/PDF 解析、异步项目 RAG 队列。

## 3. 文件生成与业务 Skill

- [ ] Word 正式模板。
  - 当前状态：后补。
  - 目标：文件生成能套用公司正式 Word 模板，而不是无模板 tracer bullet。
  - 通过：生成文件符合模板版式、占位字段正确替换。
  - 失败：版式错乱、占位符残留，或模板缺失时报原始错误。
- [ ] Excel 正式模板。
  - 当前状态：后补。
  - 目标：U03 等业务 Skill 能套用正式 Excel 模板。
  - 通过：生成 `.xlsx` 可正常打开，字段位置与业务样例一致。
  - 失败：文件损坏、字段错位，或仍是临时无模板结构。
- [ ] PPT 文件生成。
  - 当前状态：后补。
  - 目标：在 Word 链路稳定后扩展 PPT 输出。
  - 通过：能通过对话触发基础 PPT 生成并下载。
  - 失败：只返回文本，或生成文件无法打开。
- [ ] Skill 运行详情页或详情面板。
  - 当前状态：后补。
  - 目标：用户能查看 Skill 当前缺哪些字段、已收集哪些字段、生成了什么结果。
  - 通过：运行状态清晰，不需要从聊天记录里猜。
  - 失败：状态卡片缺信息，或补参后状态不更新。

## 4. 管理员维护能力

- [ ] 审计日志导出。
  - 当前状态：后补。
  - 目标：管理员可按筛选条件导出审计日志文件。
  - 通过：导出文件包含时间、用户、动作、是否成功、token 消耗等字段。
  - 失败：导出失败、字段缺失，或普通用户可导出。
- [ ] 模板上传与替换。
  - 当前状态：后补。
  - 目标：管理员可维护 Word/Excel 模板，不需要手动放文件。
  - 通过：上传、替换、查看模板状态都可用，并有权限校验。
  - 失败：模板替换后业务 Skill 仍使用旧模板，或普通用户可修改。
- [ ] 知识审核批量处理。
  - 当前状态：后补。
  - 目标：审核队列多时可批量通过、驳回或筛选。
  - 通过：批量操作有确认和结果反馈。
  - 失败：误处理、无撤销提示，或失败项无法定位。

## 5. 设置与桌面体验

- [ ] 联网搜索真实能力。
  - 当前状态：后补。
  - 目标：Chat 工具中的联网搜索开关能触发后端真实搜索能力，并在回答中标明来源。
  - 通过：开启后回答可使用联网来源；关闭后不联网。
  - 失败：开关只是本地状态，或回答不标来源。
- [ ] 全局快捷键绑定。
  - 当前状态：后补。
  - 目标：用户自定义快捷键在 Electron 全局或窗口内按预期生效。
  - 通过：快捷键修改后立即或重启后生效，冲突时有提示。
  - 失败：设置可保存但按键无效。
- [ ] 真实并排模式。
  - 当前状态：后补。
  - 目标：右上角并排入口不只是占位，可真正并排查看两个会话或内容面板。
  - 通过：用户可打开、关闭、切换并排内容，布局不重叠。
  - 失败：入口存在但无实际能力，或小窗口布局崩溃。

## 6. 远程连接

- [ ] 钉钉 Bot。
  - 当前状态：后补，非现阶段必要实现。
  - 目标：钉钉消息可进入对话系统，AI 回复可返回钉钉。
  - 通过：钉钉中发送消息后，Project_R 后端创建或复用会话，并返回 AI 回复。
  - 失败：消息丢失、重复回复，或普通用户可伪造身份。
- [ ] 钉钉配置安全存储。
  - 当前状态：后补。
  - 目标：Bot Token 等敏感配置不进入前端明文日志或可导出文件。
  - 通过：配置只在后端或安全存储中保存，响应体不泄露 token。
  - 失败：token 出现在前端、日志、文档示例或错误响应中。
