设计气质

Workbench first

Project_R 是企业 Agent 工作台，不是营销网站。视觉重点：让用户清楚 Agent 理解了什么、正在做什么、哪里要确认、结果在哪里。

✅ 应该长这样
  • 细边框 + 留白分区，信息密度适中
  • 普通选中态统一碧落暮蓝
  • 卡片只框住「具体对象」（消息、计划、工具调用…）
  • 交互有 0.12–0.16s 轻微过渡，必要时使用弹簧曲线
❌ 不要这样做
  • 装饰性大色块、渐变球、hero 标题
  • 每个页面区块都套一层阴影卡片
  • 随手写 #3b82f6 或给 active 用绿/紫/橙
  • 大面积闪烁动画、英文俏皮 loading

颜色 Token

用法：hsl(var(--token))

所有颜色必须通过 CSS 变量取值，禁止在组件里硬编码 hex。点击右上角可切换深浅主题对照。

碧落 active vs 能力提示

统一为低饱和暮蓝体系

碧落淡蓝 — 普通选中 / 激活
工作区、会话、列表项、输入框功能按钮开启态。

--control-active-*

能力提示 — 仍属于碧落体系
用于 /query、公司预设 Skill、提示词等「本轮能力」提示，不另造亮蓝体系。

Agent 交付前自检

每次前端改动必过
  • 颜色全部走 hsl(var(--token))，无硬编码 hex
  • 普通 active 只用碧落 --control-active-*，不再使用旧浅绿或临时亮蓝
  • 字号/间距/圆角取自阶梯，无 7/9/13/18 等奇值
  • 同组图标尺寸统一，z-index 用 --layer-*
  • 卡片只用于具体对象；深浅主题都看过
  • hover/active 有 0.12–0.16s 过渡，右侧面板可用 spring 曲线；尊重 prefers-reduced-motion
  • 新视觉规则先更新 ui-design-language-Gemini.md 与可视化规格，再改 CSS
