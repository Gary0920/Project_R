# Electron 手工验收 Checklist

本文档用于 Project_R 本地 Electron 前端手工验收。每次完成前端、交互、聊天展示、设置页或管理员后台相关修改后，按本清单检查。

测试前提：

- 后端已启动：`http://127.0.0.1:8000/health` 返回 `{"status":"ok"}`。
- 前端已启动：在 `frontend/` 中运行 `bun run dev` 并打开 Electron 窗口。
- 管理员账号可用：`admin / Project_R_2026`。
- 已完成后端 `python -m unittest discover -s tests` 与前端 `bun run build`。
- 产品文档以根目录 `Project_R PRD.md`、`Project_R 开发流程.md`、`Project_R 业务工作流清单.md` 为准。

## 1. 首次引导

- [ ] 清除本地引导状态后，首次启动进入欢迎引导。
  - 通过：进入 `/onboarding`，显示 Project_R 欢迎页。
  - 失败：直接进入登录页或主界面，且没有完成过引导。
- [ ] 欢迎页按钮状态正确。
  - 通过：`查看教程`、`环境检测`、`跳过` 可点击；`从其他设备迁移`、`导入他人配置` 为禁用占位。
  - 失败：按钮无响应、禁用项可点击，或入口文字不符合当前阶段状态。
- [ ] 服务器检测可保存后端地址。
  - 通过：填写可用后端地址，检测成功后能进入登录页；之后设置保持生效。
  - 失败：检测成功但未保存，或重启后地址丢失。
- [ ] 服务器检测失败时提示清晰。
  - 通过：填写错误地址后显示连接失败提示，并允许修改地址重试。
  - 失败：页面卡死、无错误提示，或无法再次检测。
- [ ] 教程页内容可读。
  - 通过：Chat、Agent、Knowledge、Skill 四类说明清晰显示。
  - 失败：文字重叠、按钮不可点，或窗口最小尺寸下内容不可读。

## 2. 登录与主界面

- [ ] 登录成功进入主界面。
  - 通过：输入管理员账号后进入 `/app`，显示聊天工作台。
  - 失败：登录后停留在登录页、Token 未保存，或刷新后丢失登录状态。
- [ ] 登录失败提示正常。
  - 通过：输入错误密码后显示可理解错误，不泄露后端堆栈。
  - 失败：无提示、页面崩溃，或暴露敏感错误。
- [ ] 软件内窗口控制按钮可用。
  - 通过：最小化、最大化/还原、关闭按钮可用，且融入应用顶部区域。
  - 失败：按钮不可点、位置错乱，或恢复传统系统标题栏。
- [ ] Chat / Agent 空状态符合预期。
  - 通过：Chat 模式显示轻量对话入口；Agent 模式显示工作区文件目录/上下文面板。
  - 失败：两个模式空状态混淆，或 Agent 区域没有工作区资料提示。
- [ ] 会话列表、消息区、输入区在 800x600 下不重叠。
  - 通过：关键按钮可见可点，输入框不遮挡消息。
  - 失败：文本溢出、按钮被挡，或布局错位。

## 3. 会话与输入

- [ ] 新建会话可用。
  - 通过：点击新建后创建空会话，并能发送消息。
  - 失败：无响应、创建失败，或新会话未选中。
- [ ] 切换会话可恢复消息历史。
  - 通过：点击历史会话后加载对应消息。
  - 失败：消息错乱、重复、丢失，或加载一直转圈。
- [ ] 删除、归档、恢复会话入口可用。
  - 通过：对应操作后 UI 与后端状态一致。
  - 失败：前端状态和后端状态不一致，或误删其他会话。
- [ ] 输入框快捷键正常。
  - 通过：Enter 发送，Shift+Enter 换行，空消息不能发送。
  - 失败：重复发送、空消息发送，或换行行为错误。
- [ ] 发送中状态正常。
  - 通过：发送时按钮禁用或显示处理中，完成后恢复。
  - 失败：可重复点击导致重复消息，或失败后无法继续发送。
- [ ] 斜杠命令 Skill 菜单可用。
  - 通过：在输入框开头或段落首输入 `/` 后弹出 Skill 菜单，支持搜索、上下键选择、Enter 确认、Esc 关闭。
  - 失败：无法唤出、过滤失效、键盘焦点错乱，或选中后输入框内容异常。
- [ ] Skill Chip / Tag 选中状态清晰。
  - 通过：选择 Skill 后输入框中显示独立标签，光标可继续输入具体内容。
  - 失败：标签和正文混在一起无法编辑，或用户不知道当前选择了哪个 Skill。

## 4. 消息展示

- [ ] Markdown 回答可读。
  - 通过：标题、列表、引用、代码块有基本排版。
  - 失败：Markdown 符号严重影响阅读。
- [ ] 代码块复制按钮可用。
  - 通过：点击复制后内容进入剪贴板。
  - 失败：复制按钮无效，或复制内容为空。
- [ ] 生成块复制按钮可用。
  - 通过：AI 生成的邮件、通知、文案等可复制块能一键复制。
  - 失败：生成内容夹杂在普通回答中，无法快速复制。
- [ ] 文件下载卡片可用。
  - 通过：生成文件后显示卡片，点击下载成功。
  - 失败：卡片不显示、按钮无效，或下载文件损坏。
- [ ] 知识库回答来源展示不误导。
  - 通过：有来源时显示来源；暂未支持点击原文时不伪装为已闭环能力。
  - 失败：来源不可用却表现为可点击，或来源信息缺失。

## 5. 工作区与 Agent 面板

- [ ] Agent 模式显示工作区文件目录。
  - 通过：能看到当前工作区资料树，帮助判断资料是否齐全。
  - 失败：只显示通用功能入口，或文件树为空但没有解释。
- [ ] 工作区切换不影响 Chat 会话基础使用。
  - 通过：切换工作区后仍可发送消息、查看会话。
  - 失败：切换后会话丢失、消息发错上下文，或页面崩溃。
- [ ] 工作区多文件上传可用。
  - 通过：可一次选择或拖拽多个文件上传，上传后文件树刷新，显示上传人、大小、更新时间、索引状态。
  - 失败：只能单文件、上传后不刷新，或状态显示不完整。
- [ ] 100MB 单文件限制提示正确。
  - 通过：普通用户上传超过 100MB 的文件时，前端直接提示“文件超过 100MB，请联系管理员”。
  - 失败：提示不一致、上传中才失败，或没有明确联系管理员提示。
- [ ] 工作区文件删除进入回收区。
  - 通过：删除文件需确认，成功后从项目文件列表消失，并可在回收区看到。
  - 失败：直接永久删除、回收区不显示，或前后端状态不一致。
- [ ] 工作区文件权限边界正确。
  - 通过：成员只能删除自己上传的文件；管理员可删除所有文件；无权限操作有清晰提示。
  - 失败：成员可删除他人文件，或无权限按钮误导用户。
- [ ] 回收区恢复与永久删除可用。
  - 通过：管理员可恢复/永久删除；成员只能处理自己上传且自己删除的文件；原路径冲突时有提示。
  - 失败：越权恢复、永久删除无确认，或冲突覆盖已有文件。
- [ ] 项目索引状态提示正确。
  - 通过：上传、删除、恢复、永久删除后项目资料索引状态可见，可手动刷新。
  - 失败：资料变更后用户不知道索引是否需要刷新。
- [ ] 项目资料作用域正确。
  - 通过：A 项目资料只在 A 项目对话可用；B 项目和无项目会话不可召回。
  - 失败：跨项目引用资料，或未加入项目用户可看到项目文件。

## 6. 设置页

- [ ] 设置页分类正常。
  - 通过：通用、服务器、提示词、归档、Agent、Chat 工具、远程连接、教程、快捷键、管理员分类可切换。
  - 失败：分类缺失、点击无响应，或内容错位。
- [ ] 通用设置可保存昵称和头像标识。
  - 通过：保存后主界面用户信息同步更新。
  - 失败：保存失败，或刷新后丢失。
- [ ] 服务器连接设置可检测后端。
  - 通过：修改后端地址后能检测成功，并影响后续 API 请求。
  - 失败：检测结果错误，或请求仍使用旧地址。
- [ ] 提示词管理三来源可用。
  - 通过：内置提示词只读，公司预设可调用，本机自定义可新建/编辑/删除。
  - 失败：来源混淆、本机提示词上传后端，或内置提示词可被修改。
- [ ] 全局底层规则不可被普通用户切换或编辑。
  - 通过：`backend/prompt_presets/global-base-prompt.md` 作为后端强制注入规则，不出现在普通用户可切换提示词列表。
  - 失败：普通用户可在设置页编辑或关闭全局底层规则。
- [ ] Chat 工具设置状态诚实。
  - 通过：真实未接入的联网搜索等能力标记为后补或本地设置。
  - 失败：设置开关暗示能力已闭环，但后端没有真实能力。
- [ ] 远程连接只保留钉钉 Bot 后补占位。
  - 通过：钉钉配置不作为当前必验功能，不阻塞本地使用。
  - 失败：要求用户必须配置钉钉，或配置后暗示已可远程对话。

## 7. 管理员后台

- [ ] 管理员账号可看到管理员区域。
  - 通过：管理员进入设置页后能看到用户、知识库、审核、模板、审计相关区域。
  - 失败：管理员区域缺失，或数据加载失败无提示。
- [ ] 普通用户不可见管理员区域。
  - 通过：普通用户不可见且直接调用接口返回 403。
  - 失败：普通用户可见或可调用管理员接口。
- [ ] 用户管理基础操作可用。
  - 通过：用户列表、新增、禁用、角色修改、重置密码符合预期。
  - 失败：操作无反馈、权限错误，或禁用用户仍可登录。
- [ ] 知识库状态与刷新入口可用。
  - 通过：可查看状态并触发刷新；失败时有明确提示。
  - 失败：刷新卡死、状态不更新，或错误不可理解。
- [ ] 知识审核基础操作可用。
  - 通过：可查看待审核知识，并通过、驳回、修改后通过。
  - 失败：审核操作失败，或通过后没有写入正式知识库。
- [ ] 审计日志可查看。
  - 通过：管理员可按用户或日期查看近期操作。
  - 失败：审计为空但无说明，或普通用户可访问。
