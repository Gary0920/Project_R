# Project_R 内网测试与功能说明

本文给测试人员使用，目标是让同事在自己的 Windows 电脑上安装 Project_R，然后通过公司内网连接 Gary 电脑上的后端进行测试。

## 1. 本次测试环境

- 后端电脑：Gary 当前这台电脑。
- 后端地址：`http://192.168.88.39:8000`。
- 安装包：`frontend/release/Project_R-Setup-0.1.3.exe`。
- 安装包 SHA256：`9798DD3BEEDF56A845BDCCA499FE45A478F04E9DBC4B8BFEE52DB603C43D9807`。
- 管理员账号：`admin / Project_R_2026`。
- 重要说明：同事电脑只安装客户端，不需要安装 Python、Node、Bun，也不需要配置 API Key。

如果 Gary 电脑的内网 IP 变化，测试人员可以在首次启动页或设置页把后端地址改成新的地址。

## 1.1 后端 IP 要不要固定

需要固定“后端地址”，否则 DHCP 重新分配 IP 后，同事电脑上的客户端默认地址会失效。

推荐做法：

1. 优先在路由器 / DHCP 服务器里给 Gary 电脑做固定租约，把 Gary 电脑网卡的 MAC 地址绑定到 `192.168.88.39`。
2. 如果不能改路由器，再在 Gary 电脑网卡里手动设置静态 IPv4。
3. 不建议长期依赖自动 DHCP 分配出来的当前地址，因为重启路由器、换网口、租约过期或网络调整后都可能变。

手动静态 IP 时要确认这些信息来自当前网络，不要凭空填写：

- IP 地址：`192.168.88.39`
- 子网掩码：通常是 `255.255.255.0`
- 默认网关：当前网络显示为 `192.168.88.1`
- DNS：可先用公司/路由器 DNS，或沿用当前自动获取到的 DNS

固定后必须重新验证：

```powershell
ipconfig
```

同事电脑浏览器打开：

```text
http://192.168.88.39:8000/health
```

看到 `{"status":"ok"}` 才算内网后端地址可用。

## 1.2 下次重新打包标准流程

以后不要手动拼 `VITE_DEFAULT_API_BASE_URL` 和 `electron-builder` 命令，直接用脚本：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-lan-installer.ps1 -BackendUrl http://192.168.88.39:8000 -Version 0.1.3
```

脚本会做四件事：

1. 用指定后端地址构建前端。
2. 生成 Windows `setup.exe` 安装包。
3. 检查打包后的 `app.asar` 是否真的写入该后端地址。
4. 启动 `win-unpacked` 客户端做白屏烟测，确认 React 根节点已渲染，且没有 `file:///.../assets` 资源丢失。
5. 输出安装包路径、版本、大小和 SHA256。

如果后端 IP 已经固定，也可以让脚本自动检测当前主网卡 IP：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-lan-installer.ps1 -Version 0.1.3
```

每次发给测试人员前，至少确认：

- 后端已用 `scripts\start-backend-lan.ps1` 启动。
- 后端监听地址是 `0.0.0.0:8000`，不是只监听 `127.0.0.1:8000`。
- Gary 本机 `http://127.0.0.1:8000/health` 正常。
- Gary 本机 `http://192.168.88.39:8000/health` 正常。
- 同事电脑浏览器 `http://192.168.88.39:8000/health` 正常。
- Windows 防火墙已放行 TCP 8000。
- `scripts\smoke-electron-package.ps1` 已通过。
- 安装包 SHA256 已记录到本文件或发版消息里。

## 2. Gary 测试前需要做什么

1. 确认 Gary 电脑和同事电脑在同一个内网。
2. 在 Gary 电脑打开 PowerShell，进入项目根目录。
3. 启动后端：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-backend-lan.ps1
```

如果第一次给同事测试，建议用管理员身份打开 PowerShell，并执行一次：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-backend-lan.ps1 -AddFirewallRule
```

4. 在 Gary 电脑浏览器打开 `http://127.0.0.1:8000/health`，看到 `{"status":"ok"}` 表示后端本机正常。
5. 在同事电脑浏览器打开 `http://192.168.88.39:8000/health`，看到 `{"status":"ok"}` 表示内网连接正常。

如果第 5 步失败，优先检查三件事：Gary 电脑后端是否启动、Windows 防火墙是否放行 8000 端口、两台电脑是否在同一个网络。

8. 输入测试账号登录。

通过标准：

失败时记录：

- 当前网络。

## 4. 基础功能测试

### 4.1 登录

测试目的：确认账号密码和后端连接正常。

步骤：

1. 打开软件。
2. 输入 `admin`。
3. 输入 `Project_R_2026`。
4. 点击登录。

通过标准：

- 成功进入聊天主界面。
- 左侧能看到会话列表或新建会话入口。
- 没有出现后端报错或白屏。

再测一次错误密码：

1. 退出登录或重开软件。
2. 输入正确账号和错误密码。
3. 点击登录。

通过标准：

- 页面显示“账号或密码错误”一类可理解提示。
- 不显示 Python 堆栈、数据库路径、API Key 等敏感信息。

### 4.2 新建聊天和普通对话

测试目的：确认最基础的 AI 对话能用。

步骤：

1. 点击新建会话。
2. 在输入框输入：`请用三句话介绍 Project_R 是什么`。
3. 按 Enter 发送。
4. 等待 AI 回复。

通过标准：

- 用户消息显示在右侧。
- AI 回复显示在消息区。
- 发送过程中有加载状态。
- 回复完成后输入框可以继续使用。

失败记录：

- 是否创建了会话。
- 是否卡在加载。
- 是否出现错误提示。
- AI 回复是否为空。

### 4.3 会话历史

测试目的：确认聊天记录不会切换后丢失。

步骤：

1. 新建一个会话并发送一条消息。
2. 再新建第二个会话并发送一条不同消息。
3. 点击左侧第一个会话。
4. 再点击第二个会话。

通过标准：

- 每个会话显示自己的消息。
- 不出现消息串台。
- 不重复显示同一条消息。

### 4.4 会话重命名、置顶、归档、恢复

测试目的：确认侧栏会话管理可用。

步骤：

1. 在左侧会话上点击右键。
2. 选择重命名，输入一个新名字，例如 `测试会话 01`。
3. 再次右键，选择置顶。
4. 再次右键，选择归档。
5. 打开设置或归档管理入口，找到该会话并恢复。

通过标准：

- 重命名后标题立即变化。
- 置顶后会话排序变化。
- 归档后左侧不再显示。
- 恢复后重新出现在会话列表。

### 4.5 Markdown 和代码块显示

测试目的：确认 AI 回复排版可读，代码可以复制。

步骤：

1. 发送：`给我一个 Markdown 示例，包含标题、列表和一段 Python 代码`。
2. 查看 AI 回复。
3. 点击代码块复制按钮。
4. 粘贴到记事本检查内容。

通过标准：

- 标题、列表、代码块排版清楚。
- 复制按钮可用。
- 粘贴内容和代码块一致。

4. 等待通知刷新。
5. 点击标记已读。

通过标准：

- 通知面板能打开。
- 新通知能出现。
- 已读状态能保存。
- 点击通知动作不会跳错页面。

## 13. 客户端更新测试

测试目的：确认软件能发现内网新版本。

步骤：

1. 用旧版本客户端登录。
2. 等待版本检查。
3. 如果后端登记了新版本，查看更新提示。
4. 点击查看更新日志。
5. 点击下载。

通过标准：

- 有新版本时显示版本号和更新说明。
- 下载有进度。
- 下载完成后提示重启安装。
- 自动更新失败时提示联系管理员获取最新版安装包。

如果后端没有登记新版本，本项可以记录为“本轮未测，不阻塞普通功能测试”。

## 16. 常见问题

### 软件提示连接失败

先在浏览器打开 `http://192.168.88.39:8000/health`。

- 如果浏览器也打不开，说明不是客户端问题，先找 Gary 检查后端或防火墙。
- 如果浏览器能打开，但软件打不开，检查软件里的服务器地址是否写错。

### 登录失败

先确认后端连接正常，再确认账号密码。不要反复猜密码，直接记录提示并反馈。

### AI 一直不回复

可能是后端 LLM 配置、模型服务或网络出网问题。记录会话、输入内容、时间和错误提示。

### 知识库没有来源

确认是否使用了 `/query`。普通聊天不会自动查知识库。

### 上传文件失败

记录文件类型、文件大小、所在工作区和错误提示。超过 100MB 的文件第一版不作为普通上传通过项。
