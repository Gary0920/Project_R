# Project_R 员工使用教程

适用对象：日常使用 Project_R 的公司员工。

本教程面向普通员工，重点讲日常对话和知识库搜索的使用方式。

## 1. 软件定位

Project_R 是公司内部 AI 办公助手。

你可以把它理解为两个入口：

- 普通 Chat：用于生成、改写、解释、总结和轻量办公任务。
- `/query` 知识库搜索：用于查询公司知识库，并获得带来源的回答。

普通 Chat 和知识库搜索是分开的。普通 Chat 不会自动读取公司知识库。

## 2. 登录与服务器连接

1. 打开 Project_R。
2. 如果进入服务器检测页，确认地址由管理员提供。
3. 检测成功后输入账号和密码。
4. 登录后进入主聊天界面。

如果连接失败，先确认网络是否在公司内网，再联系管理员确认后端地址。

## 3. 使用普通 Chat

普通 Chat 适合不需要公司资料来源的任务。

常见用法：

- 改写文字。
- 起草邮件。
- 总结一段内容。
- 解释一个概念。
- 帮你整理思路。

示例：

```text
请把下面这段客户回复改得更正式、更简洁：……
```

普通 Chat 的回答主要来自模型能力和你本轮提供的上下文。它不会自动查询公司知识库，也不会自动生成带来源的公司资料答案。

## 4. 管理会话

你可以根据工作内容建立不同会话。

建议：

- 一个主题使用一个会话。
- 重要会话可以重命名，方便以后找到。
- 暂时不用但想保留的会话可以归档。
- 归档会话可以在设置中恢复。

这样可以减少不同工作内容互相混在一起。

## 5. 使用临时附件

如果界面提供附件入口，你可以把文件作为本轮对话上下文使用。

注意：

- 临时附件只服务当前对话。
- 上传附件不等于进入知识库。
- 普通员工不需要理解知识库后台结构。

如果你希望系统根据公司长期资料回答，请使用 `/query`。

## 6. 使用 `/query` 搜索知识库

当你需要查询公司规则、流程、标准、培训资料或内部知识时，请在问题前加 `/query`。

示例：

```text
/query 公司对书面化表达有什么要求
```

系统会查询你当前权限范围内的知识库，并尽量返回可追溯来源。

## 7. 阅读知识库回答

阅读 `/query` 回答时，优先看这些内容：

1. 结论是否回答了你的问题。
2. 来源标题是否和问题相关。
3. 来源摘要是否支持回答。
4. 是否出现资料缺口、冲突或警告。

如果答案不确定，应该把问题、回答和来源一起反馈给管理员或资料负责人。

## 8. 提问建议

普通 Chat 可以这样问：

```text
请把这段说明改得更适合发给客户：……
```

知识库搜索可以这样问：

```text
/query 请总结公司关于客户沟通记录的要求
```

更好的问题通常包含：

- 你想查什么。
- 你希望得到定义、步骤还是注意事项。
- 必要时说明使用场景。

## 9. 使用边界

- 普通 Chat 不会自动查公司知识库。
- `/query` 才是知识库搜索入口。
- 知识库回答应尽量查看来源。
- 本教程主要帮助员工完成日常对话和知识库搜索。
- 不要在聊天里粘贴 API Key、密码或其他敏感信息。

这些信息能帮助管理员更快定位问题。
