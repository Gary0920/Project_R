# Project_R 快速入门

适用对象：第一次使用 Project_R 的员工。

打开 Project_R 后，可以先从两件事开始：

- 普通 Chat：写作、改写、解释、总结、头脑风暴和轻量办公问答。
- 知识库搜索：用 `/query` 查询公司知识库，并查看回答来源。

本教程面向普通员工，只讲日常对话和知识库搜索。

## 1. 登录软件

1. 打开 Project_R。
2. 如果出现服务器检测页，确认后端地址由管理员提供。
3. 检测通过后进入登录页。
4. 输入账号和密码。
5. 点击登录。

登录成功后，你会进入聊天主界面。

## 2. 发起普通 Chat

普通 Chat 适合处理不需要引用公司资料的问题。

示例：

```text
请把这段话改得更正式一些：……
```

也可以让 AI 帮你解释、整理、翻译或起草内容。

注意：普通 Chat 不会自动查询公司知识库。如果你要问公司制度、流程、标准或培训资料，请使用 `/query`。

## 3. 使用知识库搜索

知识库搜索必须显式输入 `/query`。

示例：

```text
/query 书面化原则是什么
```

发送后，系统会查询当前允许范围内的公司知识库，并返回带来源的回答。

## 4. 查看回答来源

知识库回答通常会显示来源摘要或来源面板。

你需要重点看三件事：

- 回答是否直接回应了问题。
- 来源标题是否和问题相关。
- 是否出现资料缺口、冲突或警告提示。

如果回答没有来源，先确认自己是否使用了 `/query`。

## 5. 什么时候用 Chat，什么时候用 `/query`

| 场景 | 推荐入口 |
|---|---|
| 改写一段话 | 普通 Chat |
| 起草邮件或说明 | 普通 Chat |
| 解释一个通用概念 | 普通 Chat |
| 查询公司流程、规则、标准 | `/query` |
| 查询培训资料里的定义 | `/query` |
| 需要可追溯来源的回答 | `/query` |

## 6. 常见问题

### 软件提示连接失败

先确认电脑在公司内网，然后联系管理员确认后端地址是否正确。

### 登录失败

确认账号、密码和后端连接都正确。不要反复猜密码，直接把错误提示反馈给管理员。

### AI 一直不回复

可能是模型服务或后端连接异常。记录时间、输入内容和错误提示。

### 知识库回答没有来源

先确认输入是否以 `/query` 开头。普通 Chat 不会自动查知识库。
