# Project_R 行动项提炼 Skill

**注意**：本 Skill 从会议转录文本或纪要中提炼行动项。正式的行动项生成和保存请使用文件面板的「生成纪要与行动项」按钮或右键转录文件 → 「用此转录生成纪要」。

## 功能边界

- **在项目工作区或 CRM 工作区中**：指导用户先确认已有转录文本或会议纪要文件。如果用户提供文本到会话中，可基于该文本提炼行动项供参考。正式行动项需要通过「生成纪要与行动项」按钮生成并保存到 `05-行动项/actions-latest.md`。
- **在个人工作台中**：只能处理本轮用户提供的文本，输出可在会话中查看和下载到本地，不得提供保存到项目/CRM 工作区的选项。
- **本 Skill 不直接保存或录入 GBrain**。正式行动项需要走文件面板工作流。

## 使用流程

1. 如果用户已经打开了具体的会议文件夹（有 `transcript-latest.md` 和/或 `minutes-latest.md`），建议使用「生成纪要与行动项」按钮。
2. 如果用户只有文本但没有会议文件夹：
   - 指导用户在文件面板创建会议文件夹
   - 粘贴或上传转录文本
   - 用「生成纪要与行动项」按钮正式生成
3. 如果用户提供了现有 `actions-latest.md` 内容：
   - 帮助审阅、整理或分类现有行动项
   - 如需修改正式文件，建议用户下载后修改再上传
4. 如果用户直接把文本粘贴到会话中要求提炼行动项：
   - 你可以提炼出一个临时行动项清单
   - 提醒用户这不会自动保存，正式使用需要走文件面板流程

## 输出格式

当需要直接生成行动项草稿时，使用以下结构：

```markdown
## 行动项清单
| ID | 状态 | 优先级 | 行动项 | 负责人 | 协作人 | 截止时间 | 依赖条件 | 来源 | 待确认原因 |
|---|---|---|---|---|---|---|---|---|---|
| A1 | 待确认 | 高 | ... | ...（无则待确认） | ... | ...（无则待确认） | ... | ... | ... |

## 按负责人分组
...

## 待确认行动项
（单独列出没有明确负责人或截止时间的行动项）
```

## 注意事项

- 没有来源依据的行动项必须标记为「待确认」，并写明待确认原因（如「未在转录文本中找到明确负责人」）。
- 不要编造负责人、截止时间或行动描述。
- 正式行动项的保存和版本管理必须通过文件面板的工作流完成。
- 在个人工作台中，不要暗示可以保存到项目/CRM 或录入 GBrain。
