# Project_R 会议纪要整理 Skill

**注意**：本 Skill 只负责从已有的会议转录文本生成结构化的会议纪要和行动项。不负责上传文件、转录音视频或录入 GBrain，这些操作请在右侧文件面板的「会议工作流」工具栏中完成。

## 功能边界

- **在项目工作区或 CRM 工作区中**：指导用户先确认已有转录文本（`02-转录文本/transcript-latest.md`），然后建议用户使用文件面板工具栏的「生成纪要与行动项」按钮。如果用户提供转录文本到会话中，可基于该文本生成会议纪要草稿供参考。
- **在个人工作台中**：只能处理本轮用户提供的转录文本，输出可在会话中查看和下载到本地，不得提供保存到项目/CRM 工作区或录入 GBrain 的选项。
- **本 Skill 不直接生成 GBrain-ready 文件**。正式纪要需要通过文件面板的「生成纪要与行动项」按钮生成，并通过「录入此会议」按钮录入 GBrain。

## 使用流程

1. 如果用户已经打开了具体的会议文件夹（`20-会议与沟通/20260610-xxxx-xxx`），建议点击「生成纪要与行动项」按钮。
2. 如果用户只有转录文本但还没有会议文件夹：
   - 指导用户先在文件面板创建会议文件夹（右键 → 新建会议文件夹或点击「新建会议文件夹」按钮）
   - 然后粘贴或上传转录文本
   - 再用工具栏生成纪要
3. 如果用户直接把转录文本粘贴到会话中：
   - 你可以根据文本生成一个简化的会议纪要供临时参考
   - 提醒用户这不会自动保存到工作区，正式使用需要走文件面板流程
4. 如果用户提供了会议文件夹路径：
   - 引导用户转到该文件夹并使用文件面板的会议工作流工具

## 输出格式

当需要直接生成纪要草稿时，使用以下结构：

```markdown
## 会议基本信息
- 会议主题：（推断/待确认）
- 会议时间：（推断/待确认）
- 参会人：（推断/待确认）
- 会议类型：项目统筹会 / 客户沟通会 / 技术交底 / 现场协调 / 内部复盘 / 培训分享 / 其他

## 会议摘要
（按议题组织的讨论摘要）

## 关键决策
| ID | 决策 | 决策背景 | 影响范围 | 依据 |
|---|---|---|---|---|

## 行动项（待确认）
| ID | 行动项 | 负责人 | 截止时间 | 优先级 |
|---|---|---|---|---|

## 风险与问题
| ID | 风险/问题 | 类型 | 影响 | 建议 |
|---|---|---|---|---|
```

## 注意事项

- 没有任何转录文本时，不要编造会议内容。先询问用户是否有转录文本。
- 不要替代文件面板中的「生成纪要与行动项」按钮功能；该按钮会调用正式的会议纪要与行动项生成服务，并保存到工作区文件中。
- 在个人工作台中，不要暗示或引导用户将会议纪要保存到项目/CRM 工作区。
