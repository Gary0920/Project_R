# 规范 PDF 提炼-PM版：项目执行型知识库

## 与 A版（标准知识手册）的核心区别

| 维度 | A版 | B版（本版） |
|---|---|---|
| 目标读者 | 规范工程师、技术审核员 | 项目经理、设计、采购、QC、安装队 |
| 组织方式 | 按标准章节结构 | 按规范作用类型（A-H 八大类） |
| 覆盖范围 | 单份标准逐条整理 | 跨标准（门窗/幕墙/玻璃/五金/表面处理/安装/防水/测试/认证） |
| 每条内容 | 标准原文+参数+证据 | 8 字段执行卡片 |
| 内容策略 | 尽量保留全部条款 | 强相关→主手册，弱相关→背景附录 |
| 核心价值 | "标准写了什么" | "项目要做什么" |

## 八大规范作用类型（A-H）

输出按以下类型重新组织，不再按标准章节排序：

| 类型 | 名称 | 项目作用 |
|---|---|---|
| A | 设计输入类 / Design Input | 确定项目初始条件：风压、建筑类别、环境等级、防火、声学、热工 |
| B | 性能验证类 / Performance Verification | 判断系统是否满足要求：结构强度、水密、气密、操作力 |
| C | 材料合规类 / Material Compliance | 采购和样品阶段：铝材、玻璃、五金、胶、密封条 |
| D | 构造节点类 / Construction Details | 设计和 Shop Drawing：排水、泛水、锚固、伸缩缝 |
| E | 制造与QC类 / Manufacturing & QC | 工厂生产：尺寸公差、表面质量、涂层厚度、ITP |
| F | 测试与认证类 / Testing & Certification | PMU、实验室测试、第三方认证 |
| G | 安装与现场保护类 / Installation & Site Protection | 现场安装：安装间隙、固定方式、成品保护 |
| H | 文件与交付类 / Documentation & Delivery | 验收和竣工：标签、证书、O&M、Warranty |

## 8 字段知识卡格式

每条规范内容转换为以下 8 字段卡片：

```markdown
### 知识卡 N：主题 / Knowledge Card N: Topic
- **规范作用类型 / Function Type**: A | B | C | D | E | F | G | H
- **来源标准 / Source Standard**: 标准编号 Clause X.X.X (p. X)
- **适用场景 / Scenario**: 在什么项目情况下被用到
- **责任角色 / Responsible Role**: PM / 设计 / 采购 / QC / 安装队
- **执行动作 / Action**: 具体要做的事（可执行动作）
- **检查方法 / Verification**: 如何判断是否符合
- **证据文件 / Evidence**: 需保留的文件
- **风险后果 / Risk**: 忽略该要求的实际项目风险
- **客户沟通要点 / Client Comms**: 客户/顾问质疑时如何解释
- **保留判断 / Retention**: ✅ 主手册 / 📎 背景知识
```

## 内容过滤规则

- **保留**：80% 与铝合金门窗项目直接相关的内容
- **删除**：木窗制造细节、uPVC 制造细节、与项目管理无直接关系的材料学内容
- **弱相关**：标注为 📎 背景知识，放入附录不进入主手册
- **无法转换**：无法按 8 字段格式转换的条款直接删除

## 项目执行检查项转换规则

每条参数必须转换为检查项，说明：
1. **为什么检查** — 该参数影响什么项目结果
2. **不检查会导致什么问题** — 具体风险描述
3. **如何验证** — 检查手段（图纸/报告/证书/测试/照片）
4. **需要什么文件证明** — 文件类型和出具方
