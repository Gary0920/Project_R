# 生产中变更

## 目的
统一生产变更处理流程，提升变更效率与准确性，确保变更过程中各角色清晰协作、资料可追溯。

## 流程内容

### 基本原则

| 指南 | 说明 |
| --- | --- |
| 尽量减少变更频次 | 避免不必要的重复修改 |
| 发变更时一次性发全 | 不得将一个变更拆分为多个小项提交，只有停止部分或全部生产时才拆分 |
| 内部确认后再提交 | 变更前必须完成内部确认 |
| 同一项目同一错误不重复 | 一次变更学到的必须反映到后续所有的生产文件中 |
| 工厂在做样品时重视并主动沟通 | 提前发现、提前解决 |

### Type A：可现场解决类（3000--5000 sqm）
- 在当前流程内处理，不影响现有排期；
- 设计组确认（24h内）-- 广州团队确认（24h内）；
- 文件更新在 3 天内；
- 适用于下一批次生产。

### Type B：半天内解决类（3000--5000 sqm）
- 通常限于第一批次，半天内解决；
- 工厂提出问题 -- 设计反馈 -- 确认（2--4h 内）-- 广州确认；
- 文件更新在 3 天内；
- 适用于下一批次或现场手动调整。

### Type C：2 天内解决类（3000--5000 sqm）
- 涉及结构/材料/美观方案评估；
- 仅适用于第一或前两批次；
- 工厂反馈 -- 设计提出方案 -- 内部讨论（24h）-- 广州确认；
- 文件更新在 3 天内。

### Type D：超 2 天解决类（3000--5000 sqm）
- 复杂变更需研究；
- 工厂 -- 设计研究（约 4h）-- 广州团队决策并发出指令 -- 文件更新；
- 文件更新 + 书面指令（3 天内）；
- 如需更长时间，需提前通知项目组。

### 补充说明
- 所有变更需记录来源及责任人；
- 处理完的变更建议归档截图或文件，保持一致性与可追踪性。
