# PMU 流程

## 目的
该流程用于指导项目在进行 PMU 测试时的全流程，从类型确认、报价分解、下单准备、打样生产，到测试执行、报告审核与归档，确保每一步有据可循、环节责任明确、测试结果可追溯。

## 流程内容

### 1. 确认类型与报价范围
报价内容应包括：测试费、安装费、材料费三项明确拆分。

### 2. 确认测试要求与实验室信息
- 必须包含风压要求，必须采用 AS2047 / AS4284 测试标准；
- 与客户确认实验室选址；
- 确认实验室结构排布、试验洞口钢构方式、门窗连接结构；
- 确认实验室交付节点与排期。

### 3. 图纸确认
- 完成 PMU 图纸初稿；交由客户/工程师确认；
- 同步确认完整物料清单。

### 4. 钉钉申请流程发起
项目组在钉钉系统中填写并提交 PMU 样板申请单。

### 5. 设计师下单准备

| 阶段 | 内容 | 说明 |
| --- | --- | --- |
| 采购单 | 材料准备（铝材、玻璃、五金） | 设计师出单 |
| 加工单 | 加工资料包（物料单、加工单、图纸） | |
| 安装包 | 安装资料包（现场安装需求、备料） | |
| 工厂工艺流程 | 生产工艺、打包工艺、现场安装工艺 | 内部标准流程 |

### 6. 项目组确认付款信息
- 实验室：报价、合同、付款方式确认；
- 安装队：报价、合同、付款方式确认。

### 7. 开始加工阶段
- 设计师、项目员驻厂；PMC 双方组织评审；项目员可选性在场；
- 出现问题需：拍照记录、上传项目钉钉群、备注说明、整理归档。

### 8. 送样至实验室 + 安装准备
- 项目组负责完成出库通知流程；
- 安排送样到实验室，协调安装人员进场。

### 9. PMU 预试协调
- 若现场发现问题需立即整改；
- 无问题时，确认正式测试时间，并与客户同步。

### 10. PMU 正式测试
- 测试由第三方实验室按确认方式执行；
- 项目方应在场观察并记录现场表现。

### 11. 测试报告初稿 Review
- 内部人员初审；
- 提交客户/工程师共同审阅。

### 12. 正式报告出具与归档
- 最终报告扫描归档；
- 报告应发送给客户与工程师，并统一存入共享盘。

### 补充说明
- 测试中发现的问题需同步反馈至设计图纸，并确保修订闭环；
- 所有归档文件应统一命名、统一路径管理，便于追溯；
- 整个流程每一阶段建议截图留存，便于培训与复盘。
