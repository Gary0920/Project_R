# 会议管理制度

## 目的
规范广州分公司会议的记录、整理与资料归档方式，确保信息完整留存、任务明确传达，提高会议效率与执行力。

## 规则内容

### 通用会前准备（所有实体会议适用）
最少提前15分钟完成以下动作：

**1. 设备调试**
- 投影仪/屏幕开机测试
- 会议电脑联网并登录共享盘
- 麦克风/摄像头功能检查（远程会议需额外测试耳机）

**2. 文件就绪**
- 会议材料在公盘路径打开
- 首屏显示会议主题页（含日期/参会人/议程）

**3. 环境布置**
- 桌椅整齐（U形或圆桌布局）
- 空调调至合适温度

### 内部会议专项要求

| 材料类型 | 准备标准 |
| --- | --- |
| 常规文件 | 电子版投影 |
| 需讨论文件 | 按参会人数打印单面双份（示例：8人会议打印16份） |
| 保密材料 | 会前10分钟发放，会后立即回收 |

### 客户会议特别规范
- 准备双备份饮品：夏季冰矿泉水+热茶（1:1）；冬季热咖啡+冰矿泉水（每2小时补一次）
- 客户材料打印，电子版提前发送至客户邮箱
- 应急包配置：便携WIFI、转接头三件套、公司宣传册

### 会议记录与文件管理强制规范
核心执行标准：**全程留痕 -> 智能纪要 -> 规范归档**

**工具使用：**

| 会议类型 | 记录形式 | 工具要求 |
| --- | --- | --- |
| 所有内部会议 | 录像（画面+声音） | 钉钉/腾讯会议录制功能 |
| 所有客户会议 | 必须启用Fathom工具 | 公司统一账号（联系IT开通） |

**AI 纪要必须包含：**
- 议题清单（带时间戳）
- 决策结论（红字标注）
- 待办事项表（含负责人/截止日）
- 禁止删除"未达成共识"的讨论内容

### 文件夹创建规范
命名格式：`[YYMMDD]_[会议主题]`（示例：`240815 项目交接会议`）

### 会议记录文件夹必备内容（4项核心文件）

| 文件类型 | 格式要求 | 提交时限 |
| --- | --- | --- |
| 会议录像 | 原始未剪辑文件 | 会后立即存储 |
| AI会议纪要 | PDF版（归档用）+ 可编辑版（如.docx） | 会后24小时内 |
| 会前材料 | 终版文件（按日期排序） | 会前1小时完成 |
| 项目交接文件 | 如有则存，无则省略 | 随会议纪要同步 |

### 严禁重复存储
禁止存入的公共文件类型：标准施工图纸、通用材料表、公司合同模板。

### 会后立即执行操作清单（责任归属：最后离开会议室者）

| 操作项 | 标准动作 |
| --- | --- |
| 文件清理 | 删除公用电脑所有会议文件、清空回收站、退出登录账号 |
| 设备复位 | 投影仪关闭电源+镜头盖复位、电脑返回登录界面、麦克风静音并断电 |
| 环境整理 | 桌椅归位标识线、垃圾零残留 |

## 补充说明
- 使用的 AI 纪要工具可根据项目不同调整
- 国外会议（例如Teams会议）必须使用Fathom工具
- 推荐由会议主持人或项目负责人统一整理会议资料
- 所有会议资料建议于24小时内归档完成
