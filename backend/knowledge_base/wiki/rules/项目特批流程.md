# 项目特批流程

## 目的

为规范项目执行过程中出现的特殊情形操作方式，确保审批流程透明、责任明确、过程可追溯。

## 规则内容

### 适用范围

本流程适用于以下四类情形，项目组如遇以下情况必须提前提交审批申请，并在OA系统中选择相应内容项：

| 类型 | 使用场景说明 |
| --- | --- |
| 1. 超预算 | 项目执行过程中，实际费用将超出原批准预算，需提前发起审批 |
| 2. 替换投标方案 | 原投标方案（供应商/材料/系统等）拟被替换为新的外包或供应商；需附 comparison table |
| 3. 客户未签图提前下单 | 客户尚未签图，但因时间紧急或其他特殊原因需提前下单采购 |
| 4. 工期调整 | 原计划工期发生重大变更，需审批同意后方可调整 |

### 文件路径

- 钉钉OA流程
- Comparison Table 模版链接：`\\BOLU\08 Systems\01 Templates\BGXXXX [Live] Template\02 Financials\00 Overall`

## 补充说明

- 替换投标方案、客户未签图提前下单、工期调整，请务必抄送 Frank 与东东
- 附件说明：如为替换方案，附件中需包含 comparison table
- 未经审批私自执行，将追究责任并视情节罚款
