# 系统变更审批流程要求

## 目的

为加强项目管理规范性，确保流程有据可循，避免因擅自更改系统造成质量、安全或责任问题。

## 规则内容

1. 所有公司现有系统（包括但不限于以下内容）如需变动，必须提前走审批流程：
   - 胶条、毛条、螺丝
   - 五金件（hinges、locks、handles等）
   - 系统结构、节点、安装做法等

2. 审批责任人：**Frank**
   - 只有经过 Frank 明确审批同意后，项目方才能通知设计师、供应商等第三方修改或执行

3. 项目组不得擅自决定或通知供应商或设计师进行更改，也不得私下执行替换、修改等决定

4. 系统变更相关的审批具体操作流程，已于2025年7月4号正式发布

5. 若项目组未经审批擅自更改系统内容，公司将追究责任并视情况予以罚款
