# 下加工单阶段变更

## 目的
规范客户变更情况下的操作流程、各方责任归属及变更计费机制，确保项目执行有序，减少误操作与损失。

## 流程内容

### 1. 前提要求
所有文件必须已获得客户签字确认，才能交由设计组开始加工单操作。如遇任何变更，必须按照标准变更流程处理，不得私下修改。

### 2. 客户信息变动时的操作流程
- **口头沟通说明**：项目组需与设计组领队沟通变更背景，寻求技术支持和判断建议。
- **是否停止加工判断**：由项目组决定。如决定暂停制作，需由项目组在群内以书面通知形式告知设计组，说明原因并正式指示暂停。
- **设计组评估操作责任归属**。

### 3. 变更计费机制

| 情况 | 收费标准 | 说明 |
| --- | --- | --- |
| 变更板块 >= 3 个 | 按面积收费 | RMB 30/m2（涉及订料+加工包）；RMB 12/m2（仅影响订料） |
| 变更板块 <= 2 个 | 不收费 | 但需请设计组喝下午茶 |

### 4. 操作严禁事项
- 不得因赶工期先安排设计组下单，再后补客户确认；
- 若项目组私下操作导致后续损失，责任由项目组承担。

### 5. 特殊情况处理
如客户未签字但项目必须先下单，须经 Shak 书面同意后方可执行。后果责任由项目组+Shak 共同承担。

### 6. 如遇疑难
如遇流程不清、操作判断困难，请项目组或设计组主动找 Fox 组协助协调。如有必要，Fox 组将组织专项会议。
