# 项目收尾总结流程

## 目的
在项目完成后及时开展总结归档工作，系统性回顾项目全周期资料、财务状况及经验教训，助力公司知识沉淀与能力复盘。

## 流程内容

### 1. 开始时间
最后一批发货后开始启动总结流程。

### 2. 总结结构分三部分
- **文件归集**：收集所有 IFC 图、As Built 图、VMU/PMU 资料、样品、模具、材料清单等。
- **财务总结**：输出完整项目财务报告。
- **经验教训**：整理关键经验与问题教训。

### 3. 时间要求

| 时间节点 | 工作内容 |
| --- | --- |
| 发货后立即 | 启动总结流程 |
| 1 个月内 | 输出总结初稿并内部审阅（由 Daisy 或 Nadi 初审） |
| 3 个月内 | 完成至少 95% 的总结归档 |

### 补充说明
- 建议由项目组长牵头组织，联合设计、财务、PMC 等多方参与；
- 所有总结资料需归档至共享盘并命名规范，便于后续调用；
- 可考虑将经验教训作为培训资料或案例积累使用。
