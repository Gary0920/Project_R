# 项目财务表格付款计划

> 本文为培训/经验分享资料，不等同于公司正式制度。

## 一句话总结
项目财务追踪表格的模板更新，将"Overall"选项卡信息拆分整合到"Payment Plan"和"Summary"中，规范合同应收款、外部变更、内部变更的填写逻辑，通过该表格实时监控项目理论利润。

## 核心内容

### 理论成本计算公式
`实际收款金额 × (1 - 实时利润率)`，用于计算已收款对应的理论成本，推导理论利润。

### 外部变更（External Variation）
- 利润率：`(变更索赔额 - 变更成本) / 变更索赔额`。
- 即使变更被客户拒绝（Rejected），也必须记录其理论成本。
- 累计利润率：`累计利润额 / 合同总额`，用于衡量变更对项目整体利润的贡献。

### 内部变更（Internal Variation）
- 仅记录已发生（Locked in）的成本变化，不记录预测或猜测。
- 成本节约记为正值，成本超支记为负值。
- 累计影响：`累计内部变更成本 / 合同总额`。

### 汇总表（Summary）
- 总收入（Fully Received）：`合同累计收款 + 外部变更累计收款 + 内部变更累计收益`。
- 总成本（Full Cost）：`合同理论总成本 + 外部变更总成本 + 内部变更总成本`。
- 总利润（Full Profit）：`总收入 - 总成本`。

### 保留金（Retention）
- 上限规则：例如McNab合同规定保留金总额不得超过合同总额的10%，一旦达到上限后续付款不应再扣除保留金。
- 返还规则：根据合同条款在特定阶段可能返还一半保留金。

### 避坑要点
- 禁止留空单元格。若数值为0则填"0"或使用短划线，若信息不适用则填写"N/A"或插入批注。
- 变更被拒必须在"Rejected"列填写金额并插入批注说明被拒原因。
- 付款计划至少每月核对一次，确保收款节点与项目进度匹配。
- 过往项目亏损的主因之一就是忽视对该表格的维护和审查。
