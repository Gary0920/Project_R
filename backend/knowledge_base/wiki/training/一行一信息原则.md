# 一行一信息原则

> 本文为培训/经验分享资料，不等同于公司正式制度。

## 一句话总结
所有对外发出的书面信息（即时通讯、邮件、共享文件）都必须遵循"一行一信息，一段落一信息"的准则，核心目的是让信息接收者能快速、清晰地理解内容，减少沟通成本。

## 核心内容

### 一行一信息
一个句子只表达一个核心信息点。写完一个信息点，用句号结束，另起一行。

### 一段落一信息
当多个信息点共同解释或构成一个更大的主题时，它们组成一个段落。不同主题的段落之间，必须用空行隔开。

### 信息拆分逻辑
- **结论与原因分开**：例如，"后续批次生产要抓紧"是一个信息点（结论）。"因为之前是7天一层，现在是5天一层"是另一个信息点（原因），应另起一行。
- **整体与局部/具体分开**："整体计划不变"是一个信息点。"18层4月4日开始，19层4月9日开始"是具体细节，应另起一行。

### 软件限制下的替代方案
当软件不支持换行时，使用斜杠 `/` 作为信息分隔符。格式为：`项目编号 / 费用类别 / 具体事项 / 补充说明`。

### 表格填写规范
在Sample Register等表格中，将复合信息拆分成多行。例如玻璃配置不应写成一行长文本，而应为：
```
外层: 6mm 本体着色钢化玻璃
夹层: 12mm 空气层，黑色夹条
内层: ...
```

### 避坑要点
- 不要将背景、结论、原因、具体日期等所有信息揉合在一个长段落里。
- 任何发出去的信息都是给别人看的，必须从接收者的角度思考如何组织信息。
- 不要为了追求极致的"一行一信息"而花费2-3倍的时间编辑信息，在清晰表达和自身效率间找到平衡点。
